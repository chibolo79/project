# issue-driven-fix 스킬

## 역할
`agents/issue-fixer.md`를 읽고 지시에 따라 실행한다.

## 실행 순서

### 1단계 — 점검
`issue-fixer` 에이전트를 호출한다.
- 프롬프트: `agents/issue-fixer.md를 읽고 지시에 따라 프로젝트를 점검하세요.`

### 2단계 — 결과 보고
에이전트가 반환한 처리 결과를 사용자에게 출력한다:
- 발견된 문제 수
- 등록된 이슈 번호 목록
- 수정 완료 / 수정 불가 구분
- 수정 불가 항목은 사용자에게 수동 확인 요청

## 사용법
```
점검해줘
이슈 찾아서 고쳐줘
하네스 체크해줘
문제 파악하고 이슈 등록해줘
```

## 출력 형식
```
## 점검 결과
- 발견: N건 / 수정 완료: N건 / 수동 확인 필요: N건

| 이슈 | 제목 | 처리 |
|------|------|------|
| #N   | ...  | ✅ close / ⚠ 수동 확인 필요 |
```
