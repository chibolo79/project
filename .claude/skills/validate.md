# validate 스킬

## 역할
`agents/validator.md`의 루브릭을 적용해 산출물을 평가하고 등급을 반환한다.

## 루브릭 A — 브리핑 리포트 전용 (100점 만점)

산출물이 `reports/*.html`인 경우 이 루브릭을 사용한다.

| # | 항목 | 배점 | 평가 기준 |
|---|------|------|-----------|
| 1 | 수집 항목 완결성 | 30 | config의 10개 항목이 모두 카드에 존재하는가 (N/A 포함) |
| 2 | 수치 정확성 | 25 | 수집된 수치에 명백한 오류·단위 오기가 없는가 |
| 3 | 출처 링크 | 20 | 모든 항목에 클릭 가능한 출처 링크가 있는가 |
| 4 | HTML 구조·가독성 | 15 | KPI바·카드·동향요약 구조가 올바르게 렌더링되는가 |
| 5 | SOUL 원칙 준수 | 10 | 자체 해석 생성 없이 수집 데이터만 사용했는가 |

## 루브릭 B — 범용 문서 (100점 만점)

산출물이 브리핑 리포트가 아닌 경우 이 루브릭을 사용한다.

| # | 항목 | 배점 | 평가 기준 |
|---|------|------|-----------|
| 1 | 요구사항 충족 | 30 | 사용자가 요청한 내용을 모두 포함하는가 |
| 2 | 정확성 | 25 | 오류·오탈자·잘못된 사실이 없는가 |
| 3 | 간결성 | 20 | 불필요한 반복·중복이 없는가 |
| 4 | 구조·가독성 | 15 | 헤딩·목록·코드블록이 적절히 쓰였는가 |
| 5 | SOUL 원칙 준수 | 10 | SOUL.md의 원칙에 위배되지 않는가 |

## 등급 기준
| 점수 | 등급 | 처리 |
|------|------|------|
| 90+ | S | 완료 선언 |
| 80~89 | A | 완료 선언 |
| 70~79 | B | 완료 (개선 제안 포함) |
| 60~69 | C | 재작업 권고 |
| 60 미만 | D | 재작업 필수 |

## 출력 형식
```
## 검증 결과
- 총점: XX / 100
- 등급: X

| 항목 | 점수 | 비고 |
|------|------|------|
...

### 개선 필요 항목
- ...
```

## 사용법
대화창에 아래와 같이 입력합니다:
```
오늘 리포트 검증해줘
reports/2026-06-12.html 검증해줘
```
